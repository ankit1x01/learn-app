## The Core Insight

Most apps teach topics one by one (blocked practice).
NEET tests topics randomly across 3 subjects (interleaved).
The brain becomes automatic only through **randomized retrieval under pressure**.

```
BLOCKED (how most apps work):          INTERLEAVED (how this works):
Day 1: Krebs → Krebs → Krebs           Day 1: Krebs → Ohm's Law → Periodic Table
Day 2: Glycolysis → Glycolysis         Day 2: Glycolysis → Thermodynamics → Krebs
Day 3: ETC → ETC → ETC                 Day 3: ETC → Plant Hormones → Ohm's Law

Result: Feels good, forgets fast       Result: Feels hard, remembers forever
```

Research: Interleaved practice produces **43% better retention** at test time
(Kornell & Bjork, 2008 — UCLA)

## Memory Strength Model (FSRS Algorithm)

Each concept gets 3 numbers updated after every interaction:

```
S = Stability        (how long memory lasts — days)
D = Difficulty       (how hard this concept is for this student)
R = Retrievability   (probability student can recall RIGHT NOW)

Formula:
R(t) = e^(-t / S)   where t = days since last review

Examples:
- S=1,  reviewed today    → R = 100%   (fresh)
- S=1,  reviewed 1d ago   → R = 37%    (almost forgotten)
- S=7,  reviewed 3d ago   → R = 65%    (stable)
- S=30, reviewed 10d ago  → R = 72%    (very stable)

After each answer:
- Correct + fast   → S increases significantly (memory gets stronger)
- Correct + slow   → S increases slightly
- Wrong            → S resets lower, D increases
```

**A concept is "safe" when S > 60 days.** That means even if the student
doesn't see it for 2 months, they'll still recall it during exam.

### Difficulty Calibration by Subject

```
BIOLOGY concepts tend to:
  - High D initially (large volumes of facts, diagrams)
  - D drops fast once visual encoding done
  - Application questions (assertion-reason) keep D elevated

CHEMISTRY concepts tend to:
  - Physical Chem: moderate D, formula-heavy — benefits from derivation practice
  - Organic Chem: initially high D; drops once mechanisms click
  - Inorganic: moderate-high D for reactions; benefits from periodic table anchoring

PHYSICS concepts tend to:
  - Conceptual: moderate D once underlying principle is clear
  - Numerical: separate concept (formula) from application (problem-solving)
  - Graph-based: own encoding track — visual cortex dominant
```

## The Daily Session Builder Algorithm

### Input Parameters
```
- Student's full concept list (all NEET topics, ~1,650 concepts)
- FSRS scores for each concept (S, D, R)
- Today's available time (e.g., 3 hours)
- Exam date (days remaining)
- Current coverage % per subject
```

### Subject Weight Enforcement

```
NEET Paper Distribution → Session Must Mirror:
  Biology  : 50% of questions  → 50% of session slots
  Physics  : 25% of questions  → 25% of session slots
  Chemistry: 25% of questions  → 25% of session slots

Override rule: If a subject has critical due reviews, allow ±10% flex
Never allow any subject to fall below 15% of a session
```

### Queue Construction

Every session pulls from 4 queues simultaneously:

```
QUEUE 1: DUE REVIEWS  (R < 0.85)
├── Concepts the student learned before but are fading
├── These MUST be reviewed or they'll be lost
├── Priority: lowest R first, then by PYQ tier
└── Ratio: 35% of session

QUEUE 2: NEW CONCEPTS
├── Unseen concepts, ordered by:
│   ├── PYQ tier (Tier 1 first — see tier map above)
│   ├── Chapter dependency satisfied (prerequisites learned)
│   └── Syllabus coverage deadline pressure
├── Ratio: 30% of session
└── First exposure only — not deep practice

QUEUE 3: STRENGTHEN  (Stage 2 → Stage 3)
├── Concepts student knows but not automatic yet
├── Harder question types: application, assertion-reason, matching
├── Time pressure applied (30 sec Biology, 90 sec Physics/Chem)
└── Ratio: 25% of session

QUEUE 4: CHALLENGE MIX
├── Random cross-subject questions like real NEET paper
├── No chapter label shown — raw question only
├── Tests true pattern recognition across subjects
└── Ratio: 10% of session
```

### Interleaving Rule

```python
# NEVER serve 2 consecutive questions from same chapter
# NEVER serve 2 consecutive questions from same subject
# ENFORCE subject weight ratios across 10-question windows

def build_session_sequence(queues, total_questions):
    sequence = []
    last_chapter = None
    last_subject = None
    subject_counts = {"Biology": 0, "Physics": 0, "Chemistry": 0}
    target_ratio = {"Biology": 0.50, "Physics": 0.25, "Chemistry": 0.25}

    while len(sequence) < total_questions:
        candidates = get_next_from_all_queues()
        for concept in candidates:
            # Enforce interleaving
            if concept.chapter == last_chapter: continue
            if concept.subject == last_subject: continue
            # Enforce subject weight (soft constraint — rebalance every 10 Q)
            if is_subject_over_quota(concept.subject, subject_counts, target_ratio):
                continue
            sequence.append(concept)
            last_chapter = concept.chapter
            last_subject = concept.subject
            subject_counts[concept.subject] += 1
            break

    return sequence
```

## Multi-Modal Encoding System

A concept becomes truly automatic only when encoded in **multiple brain regions**.
Each modality activates different neural pathways = stronger, more durable memory.

```
MODALITY          BRAIN REGION ACTIVATED        BEST FOR
─────────────────────────────────────────────────────────────
Reading text      Wernicke's area (language)    Definitions, facts
Diagrams/visuals  Visual cortex + hippocampus   Processes, cycles, structures
Audio (TTS)       Auditory cortex               Sequences, mnemonics
Writing by hand   Motor cortex + frontal lobe   Formulas, reactions, equations
Teaching aloud    Broca's area + prefrontal     Understanding, not just memory
Solving MCQs      Prefrontal cortex             Application, retrieval
Emotional hooks   Amygdala + hippocampus        Any — emotion = memory glue
```

### Subject-Specific Encoding Strategies

```
BIOLOGY CONCEPTS:
  Diagrams first — always. Draw before reading.
  "Label blank diagram from memory" > any flashcard
  Mnemonics for sequences (ATP yield, kingdom features, hormone functions)
  Teach-back for processes (Krebs, meiosis, immune response)
  Comparison tables for similar concepts (mitosis vs meiosis, C3 vs C4)

  Best modalities: Diagram → MCQ → Teach-back → Assertion-Reason MCQ

CHEMISTRY — PHYSICAL:
  Formula derivation by hand (not memorization — understanding)
  Solve 3 numericals per formula before it's "learned"
  Graph interpretation: draw the graph yourself, then explain it
  Units and dimensions check on every formula

  Best modalities: Derivation → Numerical → Graph → MCQ

CHEMISTRY — ORGANIC:
  Mechanism arrows must be drawn, not just seen
  "Predict the product" > "identify the reaction"
  Reaction maps: one substrate → all possible products
  Build mental flowchart: electrophile/nucleophile → mechanism → product

  Best modalities: Mechanism drawing → Predict-product MCQ → Synthesis chain

CHEMISTRY — INORGANIC:
  Periodic table anchoring: trends → exceptions → specific elements
  Reactions memorized in groups (not individually)
  Colour, state, uses — visual association technique
  Group-wise comparison tables

  Best modalities: Trend table → Mnemonics → Reaction MCQ → Anomaly MCQ

PHYSICS — CONCEPTUAL:
  Derivation by hand (every major formula — once)
  Diagram → Identify forces/fields → Apply formula
  Thought experiments: "What happens if mass doubles?"
  Common wrong answers analyzed: WHY is it wrong?

  Best modalities: Derivation → Conceptual MCQ → Numerical → Graph

PHYSICS — NUMERICAL:
  Separate concept (formula) from application (dimensional analysis)
  Solve in symbols first, substitute numbers last
  Estimation technique: approximate → exact
  Unit consistency check as automatic habit

  Best modalities: Formula → Dimensional check → Stepped numerical → MCQ
```

### Encoding Rotation Per Concept

Each concept cycles through modalities across sessions:

```
FIRST EXPOSURE (Stage 0 → 1):
  Session 1: VISUAL primer (30-sec diagram/animation or hand-drawn)
             TEXT explanation (2-min read — annotated)
             AI asks: "In your own words?" (student types)
             One example MCQ — observe how it's tested in NEET style
             Auto-generate flashcard + one-liner
  Result: Concept enters FSRS with S=1 (reviews in 1 day)

SECOND EXPOSURE (Stage 1 → 2):
  Day 2–3:   Morning retrieval — "What do you remember about X?"
             Spaced MCQs — 3 questions, 45 sec each
             Teach-back session — explain to AI in 60 sec
             Mistake analysis — AI identifies root cause of errors
             Re-encode only the gap (not full concept again)
  Result: S increases to 3–7 days per correct answer

THIRD EXPOSURE (Stage 2 → strengthening):
  Day 4–7:   Questions served WITHOUT chapter label shown
             Time pressure: 30 sec Biology, 90 sec Physics/Chem
             Mixed with 3 other topics (interleaved)
             Assertion-Reason format (NEET's hardest MCQ style)
  Result: S grows to 10–20 days

FOURTH EXPOSURE onwards (Stage 2 → 3):
  Week 2+:   Pure retrieval — question appears, no context
             Wrong = full re-encoding sequence restarts
             Right = stability S increases, interval extends
             Feynman test via AI roleplay once per week
  Result: S grows to 30–90 days. Concept is in chitta.
```

### The 7 Memory Techniques (Applied Per Session)

```
1. SPACED REPETITION
   Tool: FSRS algorithm
   How: Each concept reviewed at optimal interval
   Why: Fighting Ebbinghaus forgetting curve scientifically

2. ACTIVE RETRIEVAL (Testing Effect)
   Tool: MCQs before re-reading
   How: Test BEFORE showing content (pre-retrieval)
   Why: Struggling to recall = stronger encoding than re-reading
   Research: Roediger & Karpicke — 50% better long-term retention

3. INTERLEAVING
   Tool: Session Builder Algorithm (subject-weighted)
   How: Mix all 3 subjects every question, enforce 50/25/25 ratio
   Why: Forces brain to identify WHICH pattern to apply
   Research: 43% better retention vs blocked practice

4. ELABORATIVE INTERROGATION
   Tool: AI Tutor popup
   How: After every new concept: "Why does this work? How is it different from X?"
   Why: Connecting to prior knowledge = deep encoding
   Example: "Why does Krebs cycle happen in mitochondria matrix specifically?"

5. DUAL CODING
   Tool: Auto-generated diagrams + text together
   How: Never show text alone. Always pair with visual representation.
   Why: Two memory traces are better than one
   Research: Paivio's Dual Coding Theory (1971, still validated)

6. CONCRETE EXAMPLES
   Tool: AI generates 2 real-world + 1 NEET PYQ example per concept
   How: Abstract → concrete anchoring → exam context
   Example: "Osmosis = raisins swelling in water = RBC in hypotonic solution"

7. GENERATION EFFECT
   Tool: Fill-in-blank before showing full content
   How: Student writes/types before answer revealed
   Why: Generating information = stronger than receiving it
   Research: Slamecka & Graf (1978)
```

## NEET-Specific Challenge Modes

### Assertion-Reason Track (NEET's hardest format)

```
Introduced at Stage 2+ only. Format:
  Assertion (A): [statement]
  Reason (R):    [statement]
  Options:
    (a) Both A and R are true, R is correct explanation of A
    (b) Both A and R are true, R is NOT correct explanation
    (c) A is true, R is false
    (d) A is false, R is true

Algorithm adds A-R questions when:
  - Concept has been answered correctly 3+ times in standard MCQ
  - Stage 2 confirmed (R > 0.7, S > 5 days)
  - Related concept pair exists in same chapter

Biology A-R examples trigger higher Stage 3 conversion than standard MCQ
(requires deeper relational understanding, not surface recall)
```

If student solves them, Stage 3 is confirmed. S += bonus.
